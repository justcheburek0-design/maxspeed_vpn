# Участие в разработке MaxSpeedVPN

Спасибо за ваш интерес к участию в разработке MaxSpeedVPN! Этот документ содержит рекомендации по внесению вклада в проект.

## Кодекс поведения

- Будьте уважительны и инклюзивны
- Сосредоточьтесь на конструктивной обратной связи
- Уважайте разные точки зрения и опыт

## Как внести вклад

### Сообщение об ошибках

1. Проверьте, не была ли ошибка уже сообщена
2. Создайте новый issue с понятным заголовком и описанием
3. Включите шаги для воспроизведения ошибки
4. Укажите информацию о вашем устройстве и версии приложения
5. Приложите скриншоты или логи, если применимо

### Предложение новых функций

1. Проверьте, не было ли предложение уже внесено
2. Создайте новый issue с понятным заголовком и описанием
3. Объясните сценарий использования и ожидаемое поведение
4. Учитывайте влияние на существующий функционал

### Pull Requests

1. Форкните репозиторий
2. Создайте ветку для функции: `git checkout -b feature/my-feature`
3. Внесите изменения
4. Напишите или обновите тесты
5. Убедитесь, что все тесты проходят: `flutter test`
6. Убедитесь, что код отформатирован: `flutter format .`
7. Убедитесь отсутствии проблем анализа: `flutter analyze`
8. Коммитьте с понятными сообщениями
9. Запушьте в ваш форк
10. Создайте pull request

## Настройка окружения разработки

1. Следуйте [Руководству по настройке](SETUP.md) для настройки среды разработки
2. Создайте ветку для ваших изменений
3. Внесите изменения
4. Тщательно протестируйте
5. Отправьте pull request

## Стандарты кодирования

### Код на Dart

- Следуйте [Руководству по стилю Dart](https://dart.dev/effective-dart/style)
- Используйте `flutter format .` для форматирования кода
- Используйте осмысленные имена переменных и функций
- Добавляйте документационные комментарии для публичных API
- Держите функции маленькими и сфокусированными
- Используйте `const`-конструкторы, где возможно

### Организация файлов

```
lib/
  core/           # Основные утилиты, константы, ошибки, тема
  data/           # Модели, репозитории, источники данных
  domain/         # Сущности, use cases, интерфейсы репозиториев
  presentation/   # Экраны, виджеты, провайдеры, тема
```

### Соглашения по именованию

- **Файлы**: `snake_case.dart`
- **Классы**: `PascalCase`
- **Функции**: `camelCase`
- **Переменные**: `camelCase`
- **Константы**: `camelCase` (или `SCREAMING_SNAKE_CASE` для настоящих констант)
- **Приватные члены**: `_сПрефиксомПодчёркивания`

### Тестирование

- Пишите модульные тесты для всех use cases
- Пишите виджет-тесты для всех экранов
- Пишите интеграционные тесты для критических сценариев
- Стремитесь к покрытию кода >80%

## Сообщения коммитов

Используйте понятные, описательные сообщения коммитов:

```
feat: добавлена поддержка VLESS Reality
fix: исправлена проблема таймаута подключения
docs: обновлена документация API
refactor: упрощена логика парсинга серверов
test: добавлены тесты для парсера подписок
```

## Процесс ревью

1. Все изменения требуют как минимум одного ревью
2. Оперативно отвечайте на комментарии ревью
3. Держите pull request сфокусированными и небольшими
4. Перебазируйте на main перед слиянием
