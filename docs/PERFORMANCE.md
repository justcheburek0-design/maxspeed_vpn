# Производительность — MaxSpeed VPN

## Время запуска

| Фаза | Цель | Примечания |
|------|------|------------|
| Cold start | < 1.2 с | Dart VM + первый кадр |
| VPN handshake | < 3.0 с | TLS + обмен ключами |
| Список серверов | < 0.5 с | Кэш + скелетоны |

Профилирование: `flutter run --profile`

### Оптимизация запуска
- Откладывайте неважную инициализацию после первого кадра
- Используйте `compute()` для JSON-декодирования
- Встройте начальный список серверов в ассеты

## Память

| Компонент | Размер |
|-----------|--------|
| Flutter engine | ~30 МБ |
| Dart VM | ~15 МБ |
| Кэш серверов | ~2 МБ |
| VPN буфер | ~5 МБ |
| **Итого** | **~52 МБ** |

Следите за `AnimationController` (не утрачивайте при dispose), очищайте таймеры, не кэшируйте баннеры в полном размере.

## Батарея

Цель: ≤ 3% в час при активном туннеле.

Потребляют энергию: keep-alive heartbeat (30 с), шифрование пакетов, foreground service.

Советы:
- Режим «Экономия батареи» увеличивает keep-alive до 60 с
- Выбирайте ближайшие серверы
- Отключайтесь на доверенном Wi-Fi

## Оптимизация (для разработчиков)

1. `flutter build apk --obfuscate --split-debug-info` — tree-shaking
2. SVG вместо растровых иконок
3. Реиспользуйте вызовы MethodChannel
4. Debounce UI-событий (300 мс)
5. Lazy-load премиум-фич через deferred imports
6. Регулярный профилинг через DevTools flame chart

## Бенчмарки

```bash
flutter drive --target=test_driver/benchmark.dart --profile
```

Результаты в `benchmarks/results.json`. Регрессия > 10% — предупреждение, > 25% — падение сборки.
